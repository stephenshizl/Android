原文

         http://developer.android.com/guide/appendix/media-formats.html

 

本文档描述了Android平台支持的媒体播放解码器、网络协议。

作为一名应用软件开发者，你可以自由使用平台上的媒体解码器，这些编解码可以是平台提供，或者是特定设备支持。然而，最好习惯于使用媒体编码配置信息使得应用软件代码编写能够与具体硬件无关。

 

 

网络协议(Network Protocols)

         可以支持下述网络协议音视频的播放：

l  实时流传输协议RTSP (实时传输协议RTP, 会话描述协议SDP)

l  Http流式传输（译者注：串流传输分两种：progressive streaming和real time streaming。progressive好处是透过HTTP来传输，品质较好，只能单点播送(Unitcast)，参见文章这里。）

l  HTTP Live Streaming协议(Android 3.0及以上) （译者注：HTTP Live Streaming最初是由苹果公司针对其iPhone、iPod、iTouch和iPad等移动设备而开发的流媒体协议，点这里下载。）

注意：暂时不支持HTTPS

         对于通过HTTP或RTSP传输视频数据还有额外要求：

l  对于3GPP和MPEG-4，moov单元必须先于任何mdat原子。

l  对于3GPP、MPEG-4和WebM，音频和视频对应在同一时间偏差可能不到500KB就分开。为了尽量减少这种音频/视频偏差，可以考虑较小的在较小的块中交叉音频和视频。
